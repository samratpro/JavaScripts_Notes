## 1. What Are We Learning?

**Electron.js** is a framework for building native-feeling desktop applications using web technologies (HTML, CSS, JavaScript). Apps you already use — VS Code, Slack, Discord, WhatsApp Desktop, Figma Desktop — are built with Electron.

**React** is a JavaScript library for building user interfaces from reusable components.

When we combine them:

- **React** handles the user interface inside the app window.
- **Electron** wraps that UI in a native desktop window and gives it access to the operating system (files, menus, notifications, etc.).

**End result:** A single codebase that runs as a standalone app on Windows, macOS, and Linux.

## 3. How Electron Works (Mental Model)

Every Electron app has **two types of processes**:

| Process | What It Is | Analogy |
|---|---|---|
| **Main process** | A Node.js program. Controls windows, menus, OS-level features. One per app. | The "manager" |
| **Renderer process** | A Chromium browser tab that renders your UI (your React app lives here). | The "worker/display" |

They run in **separate memory spaces** and cannot directly share variables. They talk to each other through a system called **IPC (Inter-Process Communication)**.

```
┌────────────────────────────────────────────┐
│               MAIN PROCESS                 │
│   (Node.js — full OS access)               │
│   - Creates windows                        │
│   - Reads/writes files                     │
│   - Shows native dialogs                   │
└────────────┬───────────────────────────────┘
             │ IPC messages
             ▼
┌────────────────────────────────────────────┐
│            RENDERER PROCESS                │
│   (Chromium tab — sandboxed)               │
│   - Your React app runs here               │
│   - HTML / CSS / JS UI                     │
└────────────────────────────────────────────┘
```

**Golden rule:** The main process has full OS power. The renderer should be treated like any browser page — it should *not* get raw OS access. Communication happens through a secure bridge called the **preload script**.

## 5. Lesson 1: Your First Electron App

**Goal:** Open a native window that displays an HTML page.

### Step 1 — Create `main.js` (the main process)

```javascript
// main.js
const { app, BrowserWindow } = require('electron');
const path = require('path');

function createWindow() {
  const win = new BrowserWindow({
    width: 900,
    height: 650,
    webPreferences: {
      contextIsolation: true,
      nodeIntegration: false,
    },
  });

  // Load an HTML file into the window
  win.loadFile('index.html');
}

// Wait until Electron is ready, then open a window
app.whenReady().then(createWindow);

// Quit the app when all windows are closed (except on macOS)
app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') app.quit();
});

// On macOS, re-open a window when the dock icon is clicked
app.on('activate', () => {
  if (BrowserWindow.getAllWindows().length === 0) createWindow();
});
```

### Step 2 — Create `index.html` (what the window shows)

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>Hello Electron</title>
  </head>
  <body>
    <h1>Hello, Electron!</h1>
    <p>This is a native desktop window rendering HTML.</p>
  </body>
</html>
```

### Step 3 — Update `package.json`

```json
{
  "name": "hello-electron-react",
  "version": "1.0.0",
  "main": "main.js",
  "scripts": {
    "start": "electron ."
  },
  "devDependencies": {
    "electron": "^30.0.0"
  }
}
```

### Step 4 — Run it

```bash
npm start
```

A native window opens showing your HTML. Congratulations — you have a desktop app.

## 6. Lesson 2: Adding React

So far we rendered plain HTML. Let's replace that with a React app.

### Option A — Quick start with Vite (recommended)

```bash
npm create vite@latest renderer -- --template react
cd renderer
npm install
npm run build
cd ..
```

This creates a `renderer/` subfolder with a React project. When you run `npm run build` inside it, Vite outputs static files into `renderer/dist/`.

Update `main.js` to load the built React page:

```javascript
win.loadFile('renderer/dist/index.html');
```

### Option B — Development mode with hot reload

During development, Vite runs a dev server (e.g. `http://localhost:5173`). Update `main.js` to load from the dev server when in development:

```javascript
const isDev = !app.isPackaged;

if (isDev) {
  win.loadURL('http://localhost:5173');
} else {
  win.loadFile('renderer/dist/index.html');
}
```

Add convenient scripts to the root `package.json`:

```json
{
  "scripts": {
    "dev:renderer": "cd renderer && npm run dev",
    "dev:electron": "electron .",
    "build:renderer": "cd renderer && npm run build",
    "start": "npm run build:renderer && electron ."
  }
}
```

In one terminal run `npm run dev:renderer`, in another run `npm run dev:electron`. You now have hot-reloading React inside Electron.

## 7. Lesson 3: React Fundamentals

Before building more, let's cover the React concepts students need.

### 7.1 Components

A **component** is a JavaScript function that returns JSX (HTML-like syntax).

```jsx
// Greeting.jsx
function Greeting() {
  return <h1>Hello, students!</h1>;
}

export default Greeting;
```

### 7.2 Props (passing data in)

```jsx
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// Usage
<Greeting name="Samrat" />
<Greeting name="Priya" />
```

### 7.3 State with `useState`

State is data that changes over time and causes the UI to re-render.

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}
```

### 7.4 Side effects with `useEffect`

Run code after render — e.g., fetch data, set up a timer.

```jsx
import { useEffect, useState } from 'react';

function Clock() {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    const id = setInterval(() => setTime(new Date()), 1000);
    return () => clearInterval(id); // cleanup
  }, []);

  return <p>The time is {time.toLocaleTimeString()}</p>;
}
```

### 7.5 Lists and keys

```jsx
function TodoList({ items }) {
  return (
    <ul>
      {items.map((item) => (
        <li key={item.id}>{item.text}</li>
      ))}
    </ul>
  );
}
```

### 7.6 Forms and controlled inputs

```jsx
import { useState } from 'react';

function NameForm() {
  const [name, setName] = useState('');

  function handleSubmit(e) {
    e.preventDefault();
    alert(`Hello, ${name}`);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Your name"
      />
      <button type="submit">Greet</button>
    </form>
  );
}
```

### 7.7 Conditional rendering

```jsx
function Login({ user }) {
  if (!user) return <p>Please log in</p>;
  return <p>Welcome back, {user.name}</p>;
}
```

## 8. Lesson 4: Main ↔ Renderer Communication (IPC)

React (renderer) cannot directly read files or show OS dialogs. It must ask the main process. Electron provides two modules:

- `ipcMain` — used in `main.js`
- `ipcRenderer` — used in the renderer (but we expose it safely via the preload script)

### Patterns

| Pattern | Use case |
|---|---|
| `ipcRenderer.send` / `ipcMain.on` | Fire-and-forget (one-way) |
| `ipcRenderer.invoke` / `ipcMain.handle` | Request/response (recommended) |
| `webContents.send` / `ipcRenderer.on` | Main → Renderer push |

We'll use `invoke`/`handle` — it's promise-based and cleanest.

## 9. Lesson 5: The Preload Script and Context Isolation

The preload script runs **before** your web page loads. It has access to a limited set of Node.js APIs and can safely expose functions to the renderer through a protected bridge called `contextBridge`.

### Step 1 — Create `preload.js`

```javascript
// preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('api', {
  getAppVersion: () => ipcRenderer.invoke('app:getVersion'),
  readFile: (path) => ipcRenderer.invoke('fs:readFile', path),
  saveFile: (path, data) => ipcRenderer.invoke('fs:saveFile', { path, data }),
});
```

Now `window.api.getAppVersion()` is available in the React app — but nothing else.

### Step 2 — Wire up the preload in `main.js`

```javascript
const win = new BrowserWindow({
  width: 900,
  height: 650,
  webPreferences: {
    preload: path.join(__dirname, 'preload.js'),
    contextIsolation: true,
    nodeIntegration: false,
  },
});
```

### Step 3 — Handle requests in the main process

```javascript
// main.js
const { ipcMain, app } = require('electron');
const fs = require('fs/promises');

ipcMain.handle('app:getVersion', () => app.getVersion());

ipcMain.handle('fs:readFile', async (event, filePath) => {
  return await fs.readFile(filePath, 'utf-8');
});

ipcMain.handle('fs:saveFile', async (event, { path: filePath, data }) => {
  await fs.writeFile(filePath, data, 'utf-8');
  return true;
});
```

### Step 4 — Use it from React

```jsx
import { useEffect, useState } from 'react';

function AppVersion() {
  const [version, setVersion] = useState('...');

  useEffect(() => {
    window.api.getAppVersion().then(setVersion);
  }, []);

  return <p>App version: {version}</p>;
}
```

## 10. Lesson 6: File System, Dialogs, Menus

### 10.1 Native "Open File" dialog

```javascript
// main.js
const { dialog } = require('electron');

ipcMain.handle('dialog:openFile', async () => {
  const result = await dialog.showOpenDialog({
    properties: ['openFile'],
    filters: [{ name: 'Text', extensions: ['txt', 'md'] }],
  });
  if (result.canceled) return null;
  return result.filePaths[0];
});
```

Expose it in `preload.js`:

```javascript
contextBridge.exposeInMainWorld('api', {
  openFile: () => ipcRenderer.invoke('dialog:openFile'),
  // ... others
});
```

Use it in React:

```jsx
async function handleOpen() {
  const path = await window.api.openFile();
  if (path) console.log('User picked:', path);
}
```

### 10.2 Native notifications

```javascript
// main.js
const { Notification } = require('electron');

ipcMain.handle('notify', (event, { title, body }) => {
  new Notification({ title, body }).show();
});
```

### 10.3 Custom application menu

```javascript
// main.js
const { Menu } = require('electron');

const template = [
  {
    label: 'File',
    submenu: [
      { label: 'New', accelerator: 'CmdOrCtrl+N', click: () => {/* ... */} },
      { type: 'separator' },
      { role: 'quit' },
    ],
  },
  {
    label: 'Edit',
    submenu: [{ role: 'undo' }, { role: 'redo' }, { role: 'copy' }, { role: 'paste' }],
  },
];

Menu.setApplicationMenu(Menu.buildFromTemplate(template));
```

## 11. Lesson 7: Building a Real App — Notes Taker

Let's put everything together in a small app that lets the user write notes and save them to disk.

### `preload.js`

```javascript
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('api', {
  saveNote: (content) => ipcRenderer.invoke('note:save', content),
  loadNote: () => ipcRenderer.invoke('note:load'),
});
```

### `main.js` (handlers)

```javascript
const { app, ipcMain, dialog } = require('electron');
const fs = require('fs/promises');

ipcMain.handle('note:save', async (event, content) => {
  const { canceled, filePath } = await dialog.showSaveDialog({
    defaultPath: 'note.txt',
    filters: [{ name: 'Text', extensions: ['txt'] }],
  });
  if (canceled || !filePath) return false;
  await fs.writeFile(filePath, content, 'utf-8');
  return true;
});

ipcMain.handle('note:load', async () => {
  const { canceled, filePaths } = await dialog.showOpenDialog({
    properties: ['openFile'],
    filters: [{ name: 'Text', extensions: ['txt', 'md'] }],
  });
  if (canceled || filePaths.length === 0) return '';
  return await fs.readFile(filePaths[0], 'utf-8');
});
```

### React component — `Notes.jsx`

```jsx
import { useState } from 'react';

export default function Notes() {
  const [text, setText] = useState('');
  const [status, setStatus] = useState('');

  async function handleSave() {
    const ok = await window.api.saveNote(text);
    setStatus(ok ? 'Saved!' : 'Save cancelled');
  }

  async function handleLoad() {
    const loaded = await window.api.loadNote();
    setText(loaded);
    setStatus('Loaded');
  }

  return (
    <div style={{ padding: 20, fontFamily: 'sans-serif' }}>
      <h1>My Notes</h1>
      <textarea
        value={text}
        onChange={(e) => setText(e.target.value)}
        rows={15}
        cols={60}
        placeholder="Type your notes here..."
      />
      <div style={{ marginTop: 10 }}>
        <button onClick={handleSave}>Save</button>
        <button onClick={handleLoad} style={{ marginLeft: 10 }}>Load</button>
        <span style={{ marginLeft: 20, color: 'green' }}>{status}</span>
      </div>
    </div>
  );
}
```

### `App.jsx`

```jsx
import Notes from './Notes';
export default function App() {
  return <Notes />;
}
```

Run the app and students have a working, saveable, loadable notes editor.

## 12. Lesson 8: Packaging and Distribution

To ship the app as a `.exe` (Windows), `.dmg` (macOS), or AppImage (Linux), use **electron-builder**.

```bash
npm install --save-dev electron-builder
```

Add to `package.json`:

```json
{
  "build": {
    "appId": "com.example.notesapp",
    "productName": "Notes App",
    "files": ["main.js", "preload.js", "renderer/dist/**/*"],
    "mac": { "category": "public.app-category.productivity" },
    "win": { "target": "nsis" },
    "linux": { "target": "AppImage" }
  },
  "scripts": {
    "dist": "npm run build:renderer && electron-builder"
  }
}
```

Build installers:

```bash
npm run dist
```

Installers appear in the `dist/` folder.

## 13. Best Practices & Security

Electron apps face web-like threats *plus* desktop-like power. Teach students these rules early:

1. **Always** set `contextIsolation: true` and `nodeIntegration: false`.
2. **Never** expose `ipcRenderer` directly to the renderer. Expose a small, named API through `contextBridge`.
3. **Validate** every piece of data received through IPC in the main process — treat the renderer like an untrusted web page.
4. **Don't** load remote content into the main window. If you must, use a separate `<webview>` with strict options.
5. Keep Electron **up-to-date** to get the latest Chromium security patches.
6. Use a **Content Security Policy** in your HTML:
   ```html
   <meta http-equiv="Content-Security-Policy" content="default-src 'self'">
   ```
7. Structure code so OS-touching logic lives in the main process and UI logic in React — don't mix them.

## 14. Student Exercises

Give these to students progressively:

**Beginner**

- Modify Lesson 1 to show a window titled with the current date.
- Build a React counter with Increment, Decrement, and Reset buttons.
- Show a list of 5 fruits rendered from an array with `.map()`.

**Intermediate**

- Extend the Notes app to remember the **last opened file path** and auto-load it on startup (store the path using `app.getPath('userData')`).
- Add a word count indicator below the textarea.
- Add a custom menu item "New Note" that clears the editor.

**Advanced**

- Build a multi-file notes app: a sidebar lists all notes in a folder, clicking one loads it.
- Add a dark mode toggle that persists between app launches.
- Package the app with electron-builder and install it on your own machine.
- Add a system tray icon that keeps the app running in the background.

## Teaching Tips

- **Live code every example.** Students learn ten times more watching mistakes get fixed than reading finished code.
- **Explain the main/renderer split on day one** and revisit it every lesson. It is the #1 thing students get confused by.
- **Always keep DevTools open.** Press `Ctrl+Shift+I` (or `Cmd+Opt+I` on macOS) in a running Electron window — students should see the same console they use in the browser.
- **Version-control each lesson.** Commit at the end of every lesson so students can rewind.
- **Pair programming works well** — one student drives the main process, another the React side.

Happy teaching!
