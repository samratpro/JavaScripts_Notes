## 1. What is React?

React is a **JavaScript library** for building **user interfaces**, developed by Meta (Facebook).

- **Component-based** — UI is split into small, reusable pieces
- **Declarative** — describe *what* the UI should look like, React handles the *how*
- **Virtual DOM** — React updates only what changed, making it fast
- **One-way data flow** — data flows down from parent to child via props

### React vs Vanilla JS
| Feature | Vanilla JS | React |
|---------|-----------|-------|
| UI updates | Manual DOM manipulation | Automatic (Virtual DOM) |
| Code reuse | Copy-paste | Reusable components |
| State management | Manual | useState, useReducer |
| Scalability | Gets messy | Scales well |

## 4. JSX — JavaScript XML

JSX lets you write HTML-like syntax inside JavaScript/TypeScript.

```tsx
// JSX gets compiled to React.createElement() calls
// JSX:
const element = <h1 className="title">Hello, React!</h1>;

// Compiled to:
const element = React.createElement("h1", { className: "title" }, "Hello, React!");
```

### JSX Rules
```tsx
// 1. Return ONE root element (use fragment if needed)
function Good() {
    return (
        <div>
            <h1>Title</h1>
            <p>Paragraph</p>
        </div>
    );
}

// Use React Fragment to avoid extra div
function Better() {
    return (
        <>
            <h1>Title</h1>
            <p>Paragraph</p>
        </>
    );
}

// 2. Use className instead of class
const el = <div className="card">Content</div>;

// 3. Self-close empty tags
const img = <img src="photo.png" alt="photo" />;
const br  = <br />;

// 4. JavaScript expressions go inside { }
const name = "Samrat";
const age  = 25;

function Greeting() {
    return (
        <div>
            <h1>Hello, {name}!</h1>
            <p>Age: {age}</p>
            <p>Next year: {age + 1}</p>
            <p>{age >= 18 ? "Adult" : "Minor"}</p>
        </div>
    );
}

// 5. Styles as objects (camelCase properties)
const style = { backgroundColor: "blue", fontSize: "16px" };
const el2 = <div style={style}>Styled</div>;
// Or inline:
const el3 = <div style={{ color: "red", fontWeight: "bold" }}>Red text</div>;
```

## 5. Components — Functional

A **component** is a function that returns JSX.
- Name must start with **uppercase**
- File extension: `.tsx` (TypeScript + JSX)

```tsx
// Simple component
function Hello() {
    return <h1>Hello, World!</h1>;
}

// Arrow function component
const Welcome = () => {
    return <h2>Welcome to React!</h2>;
};

// Component with logic
function Clock() {
    const now  = new Date();
    const time = now.toLocaleTimeString();
    const date = now.toLocaleDateString();

    return (
        <div className="clock">
            <h1>{time}</h1>
            <p>{date}</p>
        </div>
    );
}

// Exporting components
export default Hello;        // default export — one per file
export { Welcome, Clock };   // named exports — multiple per file
```

### Using components
```tsx
// App.tsx
import Hello   from "./Hello";
import { Welcome, Clock } from "./components";

function App() {
    return (
        <div>
            <Hello />
            <Welcome />
            <Clock />
        </div>
    );
}

export default App;
```

## 6. Props

**Props** (properties) are how data is passed **from parent to child**.
Props are **read-only** — a child cannot modify them.

```tsx
// Define prop types with TypeScript interface
interface GreetingProps {
    name: string;
    age: number;
    isStudent?: boolean;   // optional prop
    role?: string;         // optional with default
}

// Component using props
function Greeting({ name, age, isStudent = false, role = "User" }: GreetingProps) {
    return (
        <div className="card">
            <h2>Hello, {name}!</h2>
            <p>Age: {age}</p>
            <p>Role: {role}</p>
            {isStudent && <span className="badge">Student</span>}
        </div>
    );
}

// Parent using the component
function App() {
    return (
        <div>
            <Greeting name="Samrat" age={25} isStudent={true} />
            <Greeting name="Karim"  age={22} role="Admin" />
            <Greeting name="Jamal"  age={30} />
        </div>
    );
}
```

### Props with functions (callbacks)
```tsx
interface ButtonProps {
    label: string;
    onClick: () => void;
    disabled?: boolean;
    variant?: "primary" | "secondary" | "danger";
}

function Button({ label, onClick, disabled = false, variant = "primary" }: ButtonProps) {
    return (
        <button
            onClick={onClick}
            disabled={disabled}
            className={`btn btn-${variant}`}
        >
            {label}
        </button>
    );
}

// Usage
function App() {
    const handleClick = () => alert("Button clicked!");

    return (
        <div>
            <Button label="Save"   onClick={handleClick} />
            <Button label="Delete" onClick={() => console.log("delete")} variant="danger" />
            <Button label="Submit" onClick={handleClick} disabled={true} />
        </div>
    );
}
```

## 7. Conditional Rendering

Show or hide parts of the UI based on conditions.

```tsx
interface User {
    name: string;
    isLoggedIn: boolean;
    role: "admin" | "user" | "guest";
}

function Dashboard({ user }: { user: User }) {
    // Method 1: if/else (outside JSX)
    if (!user.isLoggedIn) {
        return <p>Please log in.</p>;
    }

    return (
        <div>
            {/* Method 2: Ternary operator */}
            <h1>{user.isLoggedIn ? `Welcome, ${user.name}` : "Please log in"}</h1>

            {/* Method 3: && short-circuit (render only if true) */}
            {user.role === "admin" && <button>Admin Panel</button>}

            {/* Method 4: || fallback */}
            <p>{user.name || "Anonymous"}</p>

            {/* Method 5: switch-based helper */}
            {renderBadge(user.role)}
        </div>
    );
}

function renderBadge(role: "admin" | "user" | "guest") {
    switch (role) {
        case "admin": return <span className="badge red">Admin</span>;
        case "user":  return <span className="badge blue">User</span>;
        case "guest": return <span className="badge gray">Guest</span>;
    }
}
```

## 8. List Rendering

Render arrays of data into lists of components.

```tsx
interface Product {
    id: number;
    name: string;
    price: number;
    inStock: boolean;
}

const products: Product[] = [
    { id: 1, name: "Laptop",  price: 75000, inStock: true  },
    { id: 2, name: "Phone",   price: 25000, inStock: false },
    { id: 3, name: "Tablet",  price: 35000, inStock: true  },
];

function ProductList() {
    return (
        <ul>
            {products.map(product => (
                // key must be unique and stable — use id, not index
                <li key={product.id}>
                    <strong>{product.name}</strong> — ৳{product.price}
                    {product.inStock
                        ? <span style={{ color: "green" }}> ✅ In Stock</span>
                        : <span style={{ color: "red" }}>   ❌ Out of Stock</span>
                    }
                </li>
            ))}
        </ul>
    );
}

// Extracting to a sub-component (cleaner)
function ProductItem({ product }: { product: Product }) {
    return (
        <li key={product.id}>
            <strong>{product.name}</strong> — ৳{product.price}
        </li>
    );
}

function ProductList2() {
    return (
        <ul>
            {products.map(p => <ProductItem key={p.id} product={p} />)}
        </ul>
    );
}
```

> ⚠️ Always use a **unique `key`** prop when rendering lists. Never use array index as key if the list can change.

## 10. useState — Local State

`useState` stores and updates values inside a component. When state changes, the component **re-renders**.

```tsx
import { useState } from "react";

// Basic counter
function Counter() {
    const [count, setCount] = useState<number>(0);

    return (
        <div>
            <h2>Count: {count}</h2>
            <button onClick={() => setCount(count + 1)}>➕ Increment</button>
            <button onClick={() => setCount(count - 1)}>➖ Decrement</button>
            <button onClick={() => setCount(0)}>🔄 Reset</button>
        </div>
    );
}
```

### useState with strings and booleans
```tsx
function ToggleText() {
    const [isVisible, setIsVisible] = useState<boolean>(true);
    const [message,   setMessage]   = useState<string>("Hello!");

    return (
        <div>
            <button onClick={() => setIsVisible(!isVisible)}>
                {isVisible ? "Hide" : "Show"}
            </button>

            {isVisible && <p>{message}</p>}

            <input
                value={message}
                onChange={e => setMessage(e.target.value)}
                placeholder="Change message..."
            />
        </div>
    );
}
```

### useState with objects
```tsx
interface User {
    name: string;
    email: string;
    age: number;
}

function UserForm() {
    const [user, setUser] = useState<User>({
        name: "",
        email: "",
        age: 0,
    });

    // Update one field — spread the rest
    const handleChange = (field: keyof User, value: string | number) => {
        setUser(prev => ({ ...prev, [field]: value }));
    };

    return (
        <div>
            <input
                value={user.name}
                onChange={e => handleChange("name", e.target.value)}
                placeholder="Name"
            />
            <input
                value={user.email}
                onChange={e => handleChange("email", e.target.value)}
                placeholder="Email"
            />
            <p>Preview: {user.name} | {user.email}</p>
        </div>
    );
}
```

### useState with arrays
```tsx
function TodoList() {
    const [todos, setTodos] = useState<string[]>([]);
    const [input, setInput] = useState<string>("");

    const addTodo = () => {
        if (!input.trim()) return;
        setTodos(prev => [...prev, input]); // spread + add
        setInput("");
    };

    const removeTodo = (index: number) => {
        setTodos(prev => prev.filter((_, i) => i !== index));
    };

    return (
        <div>
            <input value={input} onChange={e => setInput(e.target.value)} />
            <button onClick={addTodo}>Add</button>
            <ul>
                {todos.map((todo, i) => (
                    <li key={i}>
                        {todo}
                        <button onClick={() => removeTodo(i)}>❌</button>
                    </li>
                ))}
            </ul>
        </div>
    );
}
```

> ⚠️ Never mutate state directly: `todos.push("x")` ❌ — always use `setTodos([...todos, "x"])` ✅

## 11. useEffect — Side Effects

`useEffect` runs code **after** the component renders. Used for: data fetching, subscriptions, timers, DOM updates.

```tsx
import { useState, useEffect } from "react";

function Timer() {
    const [seconds, setSeconds] = useState<number>(0);

    // Runs after EVERY render
    useEffect(() => {
        document.title = `Timer: ${seconds}s`;
    });

    // Runs ONCE on mount (empty dependency array)
    useEffect(() => {
        console.log("Component mounted!");
        return () => console.log("Component unmounted!"); // cleanup
    }, []);

    // Runs when 'seconds' changes
    useEffect(() => {
        console.log(`Seconds changed: ${seconds}`);
    }, [seconds]);

    // Timer with cleanup
    useEffect(() => {
        const interval = setInterval(() => {
            setSeconds(prev => prev + 1);
        }, 1000);

        return () => clearInterval(interval); // ✅ cleanup on unmount
    }, []);

    return <h2>⏱ {seconds} seconds</h2>;
}
```

### useEffect dependency rules
```tsx
useEffect(() => { /* runs after every render  */ });
useEffect(() => { /* runs ONCE on mount       */ }, []);
useEffect(() => { /* runs when a or b changes */ }, [a, b]);
```

### Fetching data with useEffect
```tsx
interface Post { id: number; title: string; }

function Posts() {
    const [posts,   setPosts]   = useState<Post[]>([]);
    const [loading, setLoading] = useState<boolean>(true);
    const [error,   setError]   = useState<string | null>(null);

    useEffect(() => {
        const fetchPosts = async () => {
            try {
                const res  = await fetch("https://jsonplaceholder.typicode.com/posts?_limit=5");
                const data: Post[] = await res.json();
                setPosts(data);
            } catch (err) {
                setError("Failed to fetch posts.");
            } finally {
                setLoading(false);
            }
        };

        fetchPosts();
    }, []); // run once on mount

    if (loading) return <p>Loading...</p>;
    if (error)   return <p style={{ color: "red" }}>{error}</p>;

    return (
        <ul>
            {posts.map(p => <li key={p.id}>{p.title}</li>)}
        </ul>
    );
}
```

## 12. useRef — DOM Reference & Persistent Values

`useRef` gives you a **mutable ref object** that does NOT trigger re-renders when changed.
Two main uses: accessing DOM elements, persisting values across renders.

```tsx
import { useRef, useEffect, useState } from "react";

// Use 1: Access a DOM element
function AutoFocusInput() {
    const inputRef = useRef<HTMLInputElement>(null);

    useEffect(() => {
        inputRef.current?.focus(); // focus on mount
    }, []);

    return <input ref={inputRef} placeholder="I am auto-focused!" />;
}

// Use 2: Store previous value without re-render
function PreviousValue() {
    const [count, setCount] = useState<number>(0);
    const prevCount = useRef<number>(0);

    useEffect(() => {
        prevCount.current = count; // update ref after each render
    }, [count]);

    return (
        <div>
            <p>Current:  {count}</p>
            <p>Previous: {prevCount.current}</p>
            <button onClick={() => setCount(c => c + 1)}>Increment</button>
        </div>
    );
}

// Use 3: Store interval/timer ID
function StopwatchRef() {
    const [running, setRunning]   = useState<boolean>(false);
    const [elapsed, setElapsed]   = useState<number>(0);
    const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);

    const start = () => {
        if (running) return;
        setRunning(true);
        intervalRef.current = setInterval(() => {
            setElapsed(prev => prev + 1);
        }, 1000);
    };

    const stop = () => {
        if (intervalRef.current) clearInterval(intervalRef.current);
        setRunning(false);
    };

    const reset = () => {
        stop();
        setElapsed(0);
    };

    return (
        <div>
            <h2>⏱ {elapsed}s</h2>
            <button onClick={start}>Start</button>
            <button onClick={stop}>Stop</button>
            <button onClick={reset}>Reset</button>
        </div>
    );
}
```

## 13. useContext — Global State

`useContext` lets you share data across the component tree **without prop drilling**.

```tsx
import { createContext, useContext, useState } from "react";

// Step 1: Define types
interface ThemeContextType {
    theme: "light" | "dark";
    toggleTheme: () => void;
}

// Step 2: Create context with default value
const ThemeContext = createContext<ThemeContextType>({
    theme: "light",
    toggleTheme: () => {},
});

// Step 3: Create provider component
function ThemeProvider({ children }: { children: React.ReactNode }) {
    const [theme, setTheme] = useState<"light" | "dark">("light");

    const toggleTheme = () => {
        setTheme(prev => prev === "light" ? "dark" : "light");
    };

    return (
        <ThemeContext.Provider value={{ theme, toggleTheme }}>
            {children}
        </ThemeContext.Provider>
    );
}

// Step 4: Create custom hook for easy access
function useTheme() {
    return useContext(ThemeContext);
}

// Step 5: Use in any child component
function Navbar() {
    const { theme, toggleTheme } = useTheme();

    return (
        <nav style={{
            background: theme === "light" ? "#fff" : "#333",
            color:      theme === "light" ? "#333" : "#fff",
            padding: "12px"
        }}>
            <span>MyApp</span>
            <button onClick={toggleTheme}>
                {theme === "light" ? "🌙 Dark" : "☀️ Light"}
            </button>
        </nav>
    );
}

function Page() {
    const { theme } = useTheme();

    return (
        <div style={{ background: theme === "light" ? "#f5f5f5" : "#222", minHeight: "100vh" }}>
            <Navbar />
            <h1>Current theme: {theme}</h1>
        </div>
    );
}

// Step 6: Wrap app with provider in main.tsx / App.tsx
function App() {
    return (
        <ThemeProvider>
            <Page />
        </ThemeProvider>
    );
}
```

## 14. useReducer — Complex State

`useReducer` is better than `useState` when state logic is **complex** or involves multiple sub-values.

```tsx
import { useReducer } from "react";

// Define state and action types
interface CartItem {
    id: number;
    name: string;
    price: number;
    quantity: number;
}

interface CartState {
    items: CartItem[];
    total: number;
}

type CartAction =
    | { type: "ADD_ITEM";    payload: CartItem }
    | { type: "REMOVE_ITEM"; payload: number }   // id
    | { type: "CLEAR_CART" };

// Reducer function — pure function, no side effects
function cartReducer(state: CartState, action: CartAction): CartState {
    switch (action.type) {
        case "ADD_ITEM": {
            const exists = state.items.find(i => i.id === action.payload.id);
            const updatedItems = exists
                ? state.items.map(i =>
                    i.id === action.payload.id
                        ? { ...i, quantity: i.quantity + 1 }
                        : i
                  )
                : [...state.items, { ...action.payload, quantity: 1 }];

            return {
                items: updatedItems,
                total: updatedItems.reduce((sum, i) => sum + i.price * i.quantity, 0),
            };
        }

        case "REMOVE_ITEM": {
            const updatedItems = state.items.filter(i => i.id !== action.payload);
            return {
                items: updatedItems,
                total: updatedItems.reduce((sum, i) => sum + i.price * i.quantity, 0),
            };
        }

        case "CLEAR_CART":
            return { items: [], total: 0 };

        default:
            return state;
    }
}

// Component using useReducer

    const [cart, dispatch] = useReducer(cartReducer, { items: [], total: 0 });
```

## 15. useMemo — Memoized Values

`useMemo` caches a **computed value** and only recalculates it when dependencies change.

```tsx
import { useState, useMemo } from "react";

function ExpensiveList() {
    const [numbers]  = useState<number[]>(Array.from({ length: 10000 }, (_, i) => i + 1));
    const [query,    setQuery]    = useState<string>("");
    const [darkMode, setDarkMode] = useState<boolean>(false);

    // Without useMemo: recalculates on EVERY render (even when darkMode changes)
    // const filtered = numbers.filter(n => n.toString().includes(query));

    // With useMemo: only recalculates when 'query' or 'numbers' changes
    const filtered = useMemo(() => {
        console.log("Recalculating filtered list...");
        return numbers.filter(n => n.toString().includes(query));
    }, [numbers, query]);

    return (
        <div style={{ background: darkMode ? "#333" : "#fff" }}>
            <button onClick={() => setDarkMode(d => !d)}>Toggle Theme</button>
            <input value={query} onChange={e => setQuery(e.target.value)} placeholder="Search numbers..." />
            <p>Found: {filtered.length} numbers</p>
            <ul>{filtered.slice(0, 20).map(n => <li key={n}>{n}</li>)}</ul>
        </div>
    );
}
```

> ✅ Use `useMemo` for **expensive calculations** — not every computation needs it.

## 16. useCallback — Memoized Functions

`useCallback` caches a **function** so it doesn't get recreated on every render.
Useful when passing functions as props to memoized child components.

```tsx
import { useState, useCallback, memo } from "react";

// Child component — wrapped with React.memo
const ItemButton = memo(({ label, onClick }: { label: string; onClick: () => void }) => {
    console.log(`Rendering: ${label}`);
    return <button onClick={onClick}>{label}</button>;
});

function Parent() {
    const [count, setCount] = useState<number>(0);
    const [text,  setText]  = useState<string>("");

    // Without useCallback: new function created every render
    // → ItemButton re-renders even when only 'text' changes

    // With useCallback: same function reference as long as deps don't change
    const handleIncrement = useCallback(() => {
        setCount(c => c + 1);
    }, []); // no deps — never recreated

    const handleReset = useCallback(() => {
        setCount(0);
    }, []); // no deps

    return (
        <div>
            <p>Count: {count}</p>
            <input value={text} onChange={e => setText(e.target.value)} />

            <ItemButton label="Increment" onClick={handleIncrement} />
            <ItemButton label="Reset"     onClick={handleReset} />
        </div>
    );
}
```

> **useMemo vs useCallback:**
> - `useMemo(() => compute(), [deps])` → caches a **value**
> - `useCallback(() => fn(), [deps])` → caches a **function**

```tsx
// hooks/useLocalStorage.ts
import { useState } from "react";

function useLocalStorage<T>(key: string, initialValue: T) {
    const [storedValue, setStoredValue] = useState<T>(() => {
        try {
            const item = window.localStorage.getItem(key);
            return item ? (JSON.parse(item) as T) : initialValue;
        } catch {
            return initialValue;
        }
    });

    const setValue = (value: T) => {
        try {
            setStoredValue(value);
            window.localStorage.setItem(key, JSON.stringify(value));
        } catch (error) {
            console.error("localStorage error:", error);
        }
    };

    return [storedValue, setValue] as const;
}
```

## 20. Controlled vs Uncontrolled Components

```tsx
import { useState, useRef } from "react";

// CONTROLLED — React controls the input value via state
function ControlledInput() {
    const [name, setName] = useState<string>("");

    return (
        <div>
            <input
                value={name}                          // state drives the value
                onChange={e => setName(e.target.value)} // state updates on change
                placeholder="Controlled input"
            />
            <p>You typed: {name}</p>
        </div>
    );
}

// UNCONTROLLED — DOM owns the value, accessed via ref
function UncontrolledInput() {
    const nameRef = useRef<HTMLInputElement>(null);

    const handleSubmit = () => {
        const value = nameRef.current?.value;
        console.log("Value:", value);
    };

    return (
        <div>
            <input ref={nameRef} defaultValue="" placeholder="Uncontrolled input" />
            <button onClick={handleSubmit}>Get Value</button>
        </div>
    );
}
```

| | Controlled | Uncontrolled |
|--|------------|--------------|
| Value stored in | React state | DOM |
| Access value via | `state` | `ref.current.value` |
| Validation | Easy (real-time) | On submit only |
| Default value | `value` prop | `defaultValue` prop |
| Use when | Most cases | File inputs, 3rd-party libs |

## 23. Error Boundaries

Error boundaries **catch JavaScript errors** in the component tree and display a fallback UI.
Must be class components (as of React 18).

```tsx
import { Component, ErrorInfo, ReactNode } from "react";

interface Props { children: ReactNode; fallback?: ReactNode; }
interface State { hasError: boolean; error: Error | null; }

class ErrorBoundary extends Component<Props, State> {
    state: State = { hasError: false, error: null };

    static getDerivedStateFromError(error: Error): State {
        return { hasError: true, error };
    }

    componentDidCatch(error: Error, info: ErrorInfo) {
        console.error("Error caught:", error, info);
    }

    render() {
        if (this.state.hasError) {
            return this.props.fallback || (
                <div style={{ color: "red", padding: "20px" }}>
                    <h2>⚠️ Something went wrong</h2>
                    <p>{this.state.error?.message}</p>
                </div>
            );
        }
        return this.props.children;
    }
}

// Usage
function App() {
    return (
        <ErrorBoundary fallback={<p>Page crashed. Please refresh.</p>}>
            <MyComponent />
        </ErrorBoundary>
    );
}
```

## 27. Custom useFetch Hook

Reusable hook for any GET request:

```tsx
// hooks/useFetch.ts
import { useState, useEffect } from "react";

interface FetchResult<T> {
    data:    T | null;
    loading: boolean;
    error:   string | null;
    refetch: () => void;
}

function useFetch<T>(url: string): FetchResult<T> {
    const [data,    setData]    = useState<T | null>(null);
    const [loading, setLoading] = useState<boolean>(true);
    const [error,   setError]   = useState<string | null>(null);
    const [trigger, setTrigger] = useState<number>(0);

    useEffect(() => {
        let cancelled = false;

        const fetchData = async () => {
            setLoading(true);
            setError(null);
            try {
                const res = await fetch(url);
                if (!res.ok) throw new Error(`HTTP ${res.status}`);
                const json: T = await res.json();
                if (!cancelled) setData(json);
            } catch (e) {
                if (!cancelled) setError(e instanceof Error ? e.message : "Error");
            } finally {
                if (!cancelled) setLoading(false);
            }
        };

        fetchData();
        return () => { cancelled = true; };
    }, [url, trigger]);

    const refetch = () => setTrigger(t => t + 1);

    return { data, loading, error, refetch };
}

// Usage in any component
interface Post { id: number; title: string; body: string; }

function PostList() {
    const { data: posts, loading, error, refetch } =
        useFetch<Post[]>("https://jsonplaceholder.typicode.com/posts?_limit=5");

    if (loading) return <p>Loading...</p>;
    if (error)   return <p>Error: {error} <button onClick={refetch}>Retry</button></p>;
    if (!posts)  return <p>No data.</p>;

    return (
        <div>
            <button onClick={refetch}>🔄 Refresh</button>
            {posts.map(p => (
                <div key={p.id}>
                    <h3>{p.title}</h3>
                    <p>{p.body}</p>
                </div>
            ))}
        </div>
    );
}
```

## 33. Context + useReducer (Mini Redux)

Full global state pattern without any external library:

```tsx
import { createContext, useContext, useReducer, ReactNode } from "react";

// Types
interface User { id: number; name: string; role: string; }

interface AppState {
    user:   User | null;
    theme:  "light" | "dark";
    count:  number;
}

type AppAction =
    | { type: "SET_USER";    payload: User }
    | { type: "LOGOUT" }
    | { type: "TOGGLE_THEME" }
    | { type: "INCREMENT" }
    | { type: "DECREMENT" };

// Reducer
const initialState: AppState = { user: null, theme: "light", count: 0 };

function appReducer(state: AppState, action: AppAction): AppState {
    switch (action.type) {
        case "SET_USER":    return { ...state, user: action.payload };
        case "LOGOUT":      return { ...state, user: null };
        case "TOGGLE_THEME":return { ...state, theme: state.theme === "light" ? "dark" : "light" };
        case "INCREMENT":   return { ...state, count: state.count + 1 };
        case "DECREMENT":   return { ...state, count: state.count - 1 };
        default:            return state;
    }
}

// Context
const AppContext = createContext<{
    state:    AppState;
    dispatch: React.Dispatch<AppAction>;
} | null>(null);

// Provider
function AppProvider({ children }: { children: ReactNode }) {
    const [state, dispatch] = useReducer(appReducer, initialState);
    return (
        <AppContext.Provider value={{ state, dispatch }}>
            {children}
        </AppContext.Provider>
    );
}

// Custom hook
function useAppContext() {
    const ctx = useContext(AppContext);
    if (!ctx) throw new Error("useAppContext must be used within AppProvider");
    return ctx;
}

// Usage in any component
function Header() {
    const { state, dispatch } = useAppContext();

    return (
        <header>
            <span>Theme: {state.theme}</span>
            <span>Count: {state.count}</span>
            <button onClick={() => dispatch({ type: "TOGGLE_THEME" })}>Toggle Theme</button>
            <button onClick={() => dispatch({ type: "INCREMENT" })}>+</button>
            {state.user
                ? <button onClick={() => dispatch({ type: "LOGOUT" })}>Logout {state.user.name}</button>
                : <button onClick={() => dispatch({ type: "SET_USER", payload: { id: 1, name: "Samrat", role: "admin" } })}>Login</button>
            }
        </header>
    );
}

// Wrap in main.tsx / App.tsx
function App() {
    return (
        <AppProvider>
            <Header />
        </AppProvider>
    );
}
```

## 34. Performance Tips

```tsx
// 1. Use React.memo to prevent unnecessary re-renders
const Child = React.memo(({ name }: { name: string }) => <p>{name}</p>);

// 2. Use useCallback for stable function references
const handleClick = useCallback(() => {
    doSomething();
}, [/* deps */]);

// 3. Use useMemo for expensive computations
const expensiveResult = useMemo(() => {
    return heavyComputation(data);
}, [data]);

// 4. Lazy load components (code splitting)
import { lazy, Suspense } from "react";

const Dashboard = lazy(() => import("./pages/Dashboard"));

function App() {
    return (
        <Suspense fallback={<p>Loading page...</p>}>
            <Dashboard />
        </Suspense>
    );
}

// 5. Use key properly in lists — use stable unique IDs
// ✅ Good
items.map(item => <Item key={item.id} {...item} />);
// ❌ Bad — index changes when list reorders
items.map((item, index) => <Item key={index} {...item} />);

// 6. Avoid anonymous functions in JSX when not needed
// ❌ Creates new function every render
<button onClick={() => handleDelete(id)}>Delete</button>
// ✅ Stable reference when wrapped in useCallback
const handleDeleteItem = useCallback(() => handleDelete(id), [id]);
<button onClick={handleDeleteItem}>Delete</button>

// 7. Batch state updates (React 18 automatic batching)
// React 18 automatically batches these into one render
const update = () => {
    setCount(c => c + 1);
    setFlag(f => !f);
    setName("Samrat");
    // Only ONE re-render total
};
```

## 35. Full Project — Task Manager App

Combining: TypeScript · useState · useReducer · useContext · useEffect · Custom Hook · React Router

```tsx
// types/index.ts
export interface Task {
    id:        number;
    title:     string;
    priority:  "low" | "medium" | "high";
    done:      boolean;
    createdAt: Date;
}

export type TaskAction =
    | { type: "ADD";    payload: Omit<Task, "id" | "createdAt" | "done"> }
    | { type: "TOGGLE"; payload: number }
    | { type: "DELETE"; payload: number }
    | { type: "CLEAR_DONE" };
```

```tsx
// context/TaskContext.tsx
import { createContext, useContext, useReducer, ReactNode } from "react";
import { Task, TaskAction } from "../types";

interface TaskContextType {
    tasks:    Task[];
    dispatch: React.Dispatch<TaskAction>;
}

const TaskContext = createContext<TaskContextType | null>(null);

function taskReducer(tasks: Task[], action: TaskAction): Task[] {
    switch (action.type) {
        case "ADD":
            return [...tasks, {
                ...action.payload,
                id: Date.now(),
                done: false,
                createdAt: new Date(),
            }];
        case "TOGGLE":
            return tasks.map(t => t.id === action.payload ? { ...t, done: !t.done } : t);
        case "DELETE":
            return tasks.filter(t => t.id !== action.payload);
        case "CLEAR_DONE":
            return tasks.filter(t => !t.done);
        default:
            return tasks;
    }
}

export function TaskProvider({ children }: { children: ReactNode }) {
    const [tasks, dispatch] = useReducer(taskReducer, []);
    return (
        <TaskContext.Provider value={{ tasks, dispatch }}>
            {children}
        </TaskContext.Provider>
    );
}

export function useTaskContext() {
    const ctx = useContext(TaskContext);
    if (!ctx) throw new Error("useTaskContext must be inside TaskProvider");
    return ctx;
}
```

```tsx
// components/AddTaskForm.tsx
import { useState } from "react";
import { useTaskContext } from "../context/TaskContext";

export function AddTaskForm() {
    const { dispatch } = useTaskContext();
    const [title,    setTitle]    = useState("");
    const [priority, setPriority] = useState<"low" | "medium" | "high">("medium");

    const handleSubmit = (e: React.FormEvent) => {
        e.preventDefault();
        if (!title.trim()) return;
        dispatch({ type: "ADD", payload: { title, priority } });
        setTitle("");
    };

    return (
        <form onSubmit={handleSubmit}>
            <input
                value={title}
                onChange={e => setTitle(e.target.value)}
                placeholder="New task..."
            />
            <select value={priority} onChange={e => setPriority(e.target.value as "low" | "medium" | "high")}>
                <option value="low">Low</option>
                <option value="medium">Medium</option>
                <option value="high">High</option>
            </select>
            <button type="submit">Add Task</button>
        </form>
    );
}
```

```tsx
// components/TaskList.tsx
import { useMemo } from "react";
import { useTaskContext } from "../context/TaskContext";
import { Task } from "../types";

const priorityColor = { low: "green", medium: "orange", high: "red" };

function TaskItem({ task }: { task: Task }) {
    const { dispatch } = useTaskContext();
    return (
        <li style={{ textDecoration: task.done ? "line-through" : "none" }}>
            <span style={{ color: priorityColor[task.priority] }}>●</span>
            {task.title}
            <button onClick={() => dispatch({ type: "TOGGLE", payload: task.id })}>
                {task.done ? "Undo" : "Done"}
            </button>
            <button onClick={() => dispatch({ type: "DELETE", payload: task.id })}>
                Delete
            </button>
        </li>
    );
}

export function TaskList() {
    const { tasks, dispatch } = useTaskContext();

    const stats = useMemo(() => ({
        total: tasks.length,
        done:  tasks.filter(t => t.done).length,
        high:  tasks.filter(t => t.priority === "high" && !t.done).length,
    }), [tasks]);

    return (
        <div>
            <p>Total: {stats.total} | Done: {stats.done} | High priority: {stats.high}</p>
            <button onClick={() => dispatch({ type: "CLEAR_DONE" })}>Clear Completed</button>
            <ul>
                {tasks.map(task => <TaskItem key={task.id} task={task} />)}
            </ul>
            {tasks.length === 0 && <p>No tasks yet. Add one above!</p>}
        </div>
    );
}
```

## 🔁 Quick Reference Summary

### Core Concepts
| Concept | Purpose |
|---------|---------|
| Component | Function that returns JSX |
| Props | Pass data from parent to child (read-only) |
| State | Internal data that triggers re-render |
| JSX | HTML-like syntax in JavaScript |
| Key | Unique identifier for list items |
| Fragment `<>` | Group elements without extra DOM node |

### Hooks
| Hook | Purpose |
|------|---------|
| `useState` | Local component state |
| `useEffect` | Side effects (fetch, timers, subscriptions) |
| `useRef` | DOM refs, persistent values (no re-render) |
| `useContext` | Consume context (global state) |
| `useReducer` | Complex state with actions |
| `useMemo` | Cache expensive computed value |
| `useCallback` | Cache function reference |
| Custom hook | Reusable stateful logic |

### React Router
| Hook / Component | Purpose |
|------------------|---------|
| `<BrowserRouter>` | Wrap app to enable routing |
| `<Routes>` + `<Route>` | Define page routes |
| `<Link>` | Navigation without reload |
| `<NavLink>` | Link with active styling |
| `useParams()` | Read URL params (`:id`) |
| `useNavigate()` | Programmatic navigation |
| `useSearchParams()` | Read/write query strings |
| `<Navigate>` | Redirect component |
| `<Outlet>` | Render nested routes |

### Performance
| Technique | When to use |
|-----------|-------------|
| `React.memo` | Prevent child re-render when props unchanged |
| `useMemo` | Cache expensive computation |
| `useCallback` | Stable function reference for memoized children |
| `lazy` + `Suspense` | Code split large components |
| Proper `key` | Stable IDs in lists, not array index |
